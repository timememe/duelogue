# DUELOGUE — Game Design Document v0.1

**Статус:** живой документ. v0.1 фиксирует решённые вопросы и оставляет «парковку» под открытые.
**Контекст исследования:** `context/rhetoric/`, `context/fighting/`.

---

## 1. Видение

DUELOGUE — карточная дуэль, в которой **реальные риторические приёмы** превращены в карты,
а ощущение поединка строится по законам файтинга (комбо, frame advantage, rage) и
театральности Ace Attorney (драматизация ключевых моментов диалога).

Цель — игра, в которой:
- любой реальный спор можно «прочитать» через её механику (карты узнаются как приёмы);
- играть **fun** — есть напряжение, comeback, rhythm, момент мастерства;
- тему колоды (Кофе, Евангелион, Драйв, любая будущая) можно менять без переписывания
  правил — меняется только фантик.

## 2. Game Loop

### Уровни цикла

| Шкала | Что происходит | Файтинговая аналогия |
|---|---|---|
| 1 ход | Карта сыграна, эффект применён | Single hit |
| 3–5 ходов | Собрано/прервано комбо | BnB |
| Половина партии | Заполнен Накал → доступен супер | Super meter |
| Партия | Срыв на низких статах → шанс comeback | Tekken Rage Art |
| Матч (3 очка) | Победа стилем | Perfect / hype finish |

### Одно очко = выход весов в ±5 (как сейчас)

Текущая система `scales_mgr` сохраняется. Не трогаем.

### Партия

1. Coin flip (как сейчас)
2. Стартовые руки (2 ATK + 1 DEF + 1 EVD)
3. Ходы: каждый игрок играет по 1 карте за ход
4. Дорожка из 3 последних карт обновляется каждый ход
5. Проверка комбо после хода игрока
6. Эвент-чек, drawn до hand limit, victory check (как сейчас)

## 3. Core mechanics

### 3.1 RPS-треугольник (без изменений)

```
       ATTACK
        ↗  ↘
   EVASION ← DEFENSE
```

`RulesEngine.get_advantage()` — рабочая основа, не трогаем.

### 3.2 Дорожка дебатов (combo track) — НОВОЕ

**Главная новая механика.** Видимая лента из последних 3 карт (общая, не своя у каждого):
- Чередование `[ТЫ][ОН][ТЫ]` или `[ОН][ТЫ][ОН]` в зависимости от того, кто начинал партию
- Каждый ход последняя карта сдвигается, старая выпадает
- Рецепты комбо «читают» окно с точки зрения **игрока** («2 хода назад **я** сыграл X»)
- При совпадении с рецептом — комбо срабатывает, появляется именная вспышка (Ace Attorney-style)

**Длина окна v0.1:** 3. Может вырасти до 5 после прототипа.

### 3.3 Четыре типа комбо-рецептов

| Тип | Пример паттерна | Чувство |
|---|---|---|
| **Соло** | `[ТЫ:LOGIC] _ [ТЫ:LOGIC]` (ход оппонента безразличен) | FG BnB |
| **Реактивный** | `[ТЫ:CANCEL] [ОН:LOGIC_ATK] [ТЫ:LOGIC_ATK]` → «Сократов капкан» | Поймал на ошибке |
| **Бейт** | `[ТЫ:EMOTION_ATK] [ОН:CANCEL] [ТЫ:?]` — следующий ход открывает «Подмену тезиса» | Манипуляция |
| **Эхо/контра** | `[ТЫ:LOGIC] [ОН:EMOTION] [ТЫ:LOGIC]` → «Холодный разум» | Стилевая контра |

### 3.4 Эффект комбо

При срабатывании рецепта применяется **эффект комбо** поверх обычного эффекта последней карты:

- Бонусный урон / heal
- Принудительный эффект на оппонента (сброс карты, дебафф)
- Получение «Накала» сверху
- В рaredко — особый стат-эффект (например, оппонент пропускает ход)

### 3.5 Damage scaling (из файтингов)

Чтобы не было оптимальной стратегии «жми один приём всю партию»:
- Повторное использование одного `effect` в партии → множитель -10% за повтор
- Сброс множителя при смене категории или после 3+ ходов без повтора
- Минимум 30% от базового урона

### 3.6 Накал (super meter)

Шкала из 3 сегментов:
- Растёт от **успешных атак** (свой ход) и от **получения урона** (компенсация)
- Тратится частями (1 сегмент = усиление следующей карты) или целиком (супер-карта)
- Видна в UI

### 3.7 Срыв (rage / comeback) — отложенный

При `logic + emotion ≤ 3` у игрока разблокируется одноразовая **Rage-карта**:
- Большой урон одного типа, одноразовая
- Один раз за партию
- Visual telegraph (оппонент видит, что ты в срыве)

Пересекается с эвентом `CRITICAL_TURNING_POINT` — связывается с ним.

### 3.8 Burst — отложенный

Одна карта за партию у каждого игрока:
- Отменяет **последние 2 хода оппонента** (не только последний, как обычный CANCEL)
- Возвращает инициативу
- Не лечит, не наносит урона — просто респайт

## 4. Архитектура: три слоя

**Критическое правило проекта.** Чтобы темы свапались без затрагивания дизайна, всё разбито на три уровня, ни один из которых не «знает» о следующем выше:

```
LAYER 1 — Engine effects (атомы)
  LOGIC, EMOTION, SHIELD, CANCEL, MIRROR, REFLECT, HEAL_*, RANDOM
  → core/, autoloads/, resources/
  → НИКОГДА не меняется при добавлении темы

LAYER 2 — Combo grammar (тактика)
  card_id, category, effect, combo_tags (starter/linker/finisher/interrupt/bait)
  combo recipes referencing layer 1
  → data/combos/, core/combo_resolver.gd (новый)
  → меняется только при ребалансе/новых механиках

LAYER 3 — Theme (фантик)
  card_name, text_variants, description, art, sfx, character_portraits
  → data/themes/<theme_name>/
  → свободно добавляются/меняются
```

### Что это значит для текущей кодовой базы

Сейчас в `data/decks/*.json` карточный контент и тема смешаны. По мере реализации:
- Универсальные карты (по `_summary_universal_methods.md`) выносятся в `data/cards/universal_cards.json`
  с `card_id`, `category`, `effect`, базовыми числами, `combo_tags`.
- Темы в `data/themes/<name>/` содержат только: `theme_name`, `card_overrides` (name/text/art per card_id).
- Колода = список `card_id` + ссылка на тему.

## 5. Колоды, темы, архетипы

### Колода
Набор `card_id` (универсальных). Колода тематически нейтральна.

### Тема
Фантик: имена/тексты/арт. Берётся из `data/themes/<name>/`.

### Архетип
Стиль игры + AI-стратегия + стартовая статистика. См. `context/rhetoric/debate_archetypes.md`
(10 архетипов: Скептик, Эмпат, Циник, Идеалист, Демагог, Софист, Прагматик, Провокатор, Стратег, Догматик).

Архетипы могут иметь:
- Уникальный **архетип-буст** (один раз за матч) — аналог V-Trigger / Instinct (см. `fighting_meter_and_super.md`)
- Уклон колоды (Демагог = больше EMOTION_ATK, Скептик = больше DEFENSE)
- Уникальные **именные комбо**, доступные только этому архетипу

### Матрица

```
колода = архетип × тема

«Скептик с темой Кофе»     → разговор за столом, холодная аналитика
«Демагог с темой Драйв»    → яростный спор за рулём
«Эмпат с темой Евангелион» → диалог Синдзи и Аски
```

## 6. Карточный набор

База — `_summary_universal_methods.md`: ~34 карты, 3 tier-а.

- **Tier 1** (8) — обязательны в каждой колоде, ядро
- **Tier 2** (16) — расширение, дифференциация архетипов
- **Tier 3** (10) — редкие, для финишей и стиля

Распределение по типам: ~19 ATK / ~7 DEF / ~7 EVD + спец.

### Combo tags

Каждая карта помечена `combo_tags: PackedStringArray`:
- `starter` — хорошо стоит в начале комбо
- `linker` — хорошо в середине
- `finisher` — хорошо в конце
- `interrupt` — годится для реактивных комбо
- `bait` — может работать как приманка
- `pressure` — для damage scaling счётчика

Эти теги — это **подсказки рецептам**, эмерджентный слой поверх жёстко прописанных именных комбо.

## 7. Список именных комбо (стартовый, v0.1)

Финальный набор — после прототипа. Стартовый каркас:

| Имя | Паттерн | Эффект |
|---|---|---|
| **Сократов капкан** | `[ТЫ:CANCEL][ОН:LOGIC_ATK][ТЫ:LOGIC_ATK]` | +2 урона, оппонент сбрасывает 1 карту |
| **Эмоциональный шквал** | `[ТЫ:EMOTION_ATK]×2 + [ТЫ:EMOTION_ATK]` (3 эмоц атаки подряд от тебя в окне) | Пробивает SHIELD, +50% урона |
| **Холодный разум** | `[ТЫ:LOGIC][ОН:EMOTION][ТЫ:LOGIC]` | Снимает у оппонента emotion-урон в logic |
| **Зеркальный финиш** | `[ТЫ:SHIELD][ОН:любая ATK][ТЫ:MIRROR]` | Возвращает оппоненту 200% урона |
| **Стальная стена** | `[ТЫ:SHIELD][ТЫ:HEAL_LOGIC]` (через ход) | Щит больше не разрушается до конца партии |
| **Серенада** | `[ТЫ:HEAL_EMOTION][ОН:?][ТЫ:EMOTION_ATK]` | Урон + heal одновременно |
| **Догматический бур** | `[ТЫ:LOGIC_ATK]×2` (одна и та же карта дважды) | Дебафф оппонента на 1 ход, но твоя следующая ATK -50% |
| **Подмена тезиса** | bait: `[ТЫ:слабая EMOTION_ATK]` ловит CANCEL → следующий ход твоя ATK +uron | Финиш «вскрытия подмены» |

8 рецептов хватит для прототипа. После плейтеста расширим до 12-15.

## 8. UI/UX направление

### Главный экран

```
┌──────────────────────────────────────────────────┐
│ TURN 7                          SCALES: +2   ⚖    │
│                                                   │
│         OPPONENT                                  │
│         [Logic 4/4] [Emotion 2/4] [●○○]           │
│         Last: [Sarcasm]                           │
│                                                   │
│ ─── ДОРОЖКА ──────────────────────────────────── │
│    [ТЫ:CANCEL]  [ОН:LOGIC]  [ТЫ:?]               │
│ ─────────────────────────────────────────────── │
│                                                   │
│         YOU                                       │
│         [Logic 3/4] [Emotion 4/4] [●●○]          │
│         НАКАЛ: ▓▓░       SHIELD: 2               │
│                                                   │
│ HAND: [card][card][card][card][card]             │
└──────────────────────────────────────────────────┘
```

### Срабатывание комбо

- Все остальные элементы UI слегка темнеют
- Рамка вокруг 3 ячеек дорожки
- Большим шрифтом название приёма
- Sting (звук) и flash
- Lingering 0.5-1 сек, потом возврат к основному экрану

### Накал

- Шкала растёт визуально с pulse-эффектом при заполнении сегмента
- При 3/3 — мигает, подсказывает, что супер доступен

### Срыв

- Когда статы упали до триггера — портрет персонажа меняется
- В колоде появляется Rage-карта с особой подсветкой

## 11. Roadmap прототипа

Минимальный плейтестируемый прототип, чтобы проверить fun-петлю до больших инвестиций:

### Этап 1: Дорожка + 5 комбо (видимая механика)
1. Сделать видимую дорожку из 3 ячеек в `game_screen.tscn`
2. Создать `core/combo_resolver.gd` — резолвер рецептов по окну
3. Жёстко описать 5 рецептов (по одному из каждого типа + 1 запас) в `data/combos/v0_combos.json`
4. Триггерить визуальную вспышку при срабатывании
5. Прогнать через `test/test_match.gd` AI vs AI — оценить частоту срабатываний

### Этап 2: Накал
6. Добавить `tension: int` в `CharacterStats`
7. UI-шкала
8. Бонус +50% к следующей карте за 1 сегмент

### Этап 3: Burst + Rage
9. Одна универсальная Burst-карта в каждой колоде
10. Одна Rage-карта на триггер `logic + emotion ≤ 3`

### Этап 4: Архитектурная чистка
11. Разделить колоды и темы в `data/`
12. Универсальный карточный пул в `data/cards/`
13. Темы в `data/themes/<name>/`

### Этап 5: Плейтест и фидбек
14. Самим поиграть на разных колодах
15. Зафиксировать ощущения в `context/playtest_notes.md`
16. v0.2 дизайн-док с правками
