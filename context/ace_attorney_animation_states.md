# Ace Attorney — справочник анимационных стейтов персонажей

**Назначение:** сборник всех известных состояний/поз/анимаций спрайтов персонажей в серии
Ace Attorney (Gyakuten Saiban), собранный из открытых источников (Court Records, Ace Attorney
Wiki / Spriters Resource, объекция-конструктор objection.lol) + доменное знание. Референс для
дизайна анимационных стейтов DUELOGUE, не привязан к текущей архитектуре проекта.

---

## 1. Базовая модель: два слоя стейта

Спрайт персонажа в AA всегда описывается **комбинацией двух независимых осей**:

1. **Эмоциональная поза (pose)** — статичная «базовая» картинка, задающая настроение сцены.
2. **Речевой оверлей (talking / idle)** — играет ли рот/анимация болтовни поверх позы.

То есть почти каждая поза существует в двух версиях:
- `Pose` (молчит, статичный кадр или мелкий idle-луп — дыхание, покачивание)
- `Pose + Talking` (тот же кадр, но с циклом открытия/закрытия рта, иногда доп. движение рук/головы)

`objection.lol` формализует это явно: у каждой позы есть флаг **"Do not talk"**, который глушит
речевой оверлей и оставляет только базовую позу.

## 2. Основные эмоциональные позы (общие для большинства персонажей)

Стандартный набор, повторяющийся почти у каждого играбельного адвоката/обвинителя/свидетеля
(терминология по Court Records / Ace Attorney Wiki галереям):

| Поза | Описание | Обычный триггер |
|---|---|---|
| **Confident / Normal** | нейтральная уверенная стойка, база по умолчанию | старт реплики, обычный диалог |
| **Confident Talking** | то же + анимация речи | во время произнесения строки |
| **Thinking / Considering** | рука у подбородка, взгляд в сторону | пауза, размышление, "hmm" |
| **Pointing** | указывает пальцем на собеседника (статично) | обвинение, конфронтация |
| **Pointing Talking** | указывает + говорит | "Hold it!"-подобные реплики, прямое обвинение |
| **Objecting / Objection pose** | резкий выброс руки вверх, часто отдельный "wipe"-кадр захода в позу | момент "Objection!"/"Take that!" |
| **Desk Slam** | удар руками/предметом по столу, отдельная короткая анимация перехода | акцентная реакция, разоблачение |
| **Damage / Shocked** | голова дёргается назад, часто с "трещиной"-эффектом экрана | получил укол/улику против себя |
| **Damage Talking** | shocked-поза + речь (оправдывается под ударом) | ответ сразу после damage |
| **Sweating / Nervous** | капля пота, поза чуть сжатая | под давлением, неуверенность |
| **Sweating Talking** | нервно говорит | оправдания, увиливание |
| **Embarrassed / Flustered** | наклон вперёд, рука на лице, легкий блеш | смущение, стыд, неловкая правда |
| **Embarrassed Talking** | то же + речь | — |
| **Disheartened / Dejected** | опущенные плечи, поза "сдулся" | проигрыш аргумента, тупик |
| **Disheartened Talking** | то же + речь | — |
| **Angry / Furious** | нахмуренные брови, сжатые кулаки | гнев, крик, отдельные "shouting"-кадры |
| **Angry Talking** | — | — |
| **Happy / Smug / Grinning** | самодовольная улыбка (часто у прокуроров-антагонистов) | триумф, издёвка |
| **Laughing** | отдельная короткая loop-анимация смеха (иногда со звуком) | издёвка, разрядка |
| **Surprised / Startled** | короткая вставка-реакция на новый факт | неожиданный твист |

## 3. Переходные/одноразовые анимации (non-loop, "one-shot")

Не идл-позы, а короткие клипы, которые проигрываются один раз и затем возвращают персонажа
в исходную/новую idle-позу:

- **Entrance / Exit** — заход/выход персонажа в кадр (свидетели, новые действующие лица).
- **Desk Slam-in / Slam-out** — переход в позу удара по столу и обратно.
- **Gavel Bang** (судья) — удар молоточком, привязан к финализации хода/вердикту.
- **Evidence Presentation** — рука с уликой выезжает в кадр (обычно отдельный UI-слой, не спрайт персонажа).
- **Pre-animations** (термин objection.lol) — короткие анимации, привязанные к конкретной позе
  конкретного персонажа перед основным кадром: например, Франциска фон Карма хлыстом бьёт по
  столу перед тем как встать в позу — уникальная для персонажа вставка, не общий стейт.
- **Screen shake / crack overlay** — не сам спрайт, а экранный эффект, который триггерится
  синхронно с Damage-позой (визуальный "удар" по игроку).
- **Blink insert** — при статичном idle кадр моргания вставляется отдельным файлом/кадром
  через случайный таймер, не отдельная "поза", а модуляция уже идущего стейта.

## 4. Ролевые вариации по типу персонажа

- **Судья (Judge):** отдельный, более узкий набор поз — Neutral, Talking, Confused/Scratching head,
  Gavel bang, Shocked/Surprised, Sleeping (комический стейт в отдельных играх), Angry (редко).
- **Свидетель (Witness):** полный набор из §2, плюс специфичные персональные тики (у каждого
  свидетеля обычно 1-2 уникальные "подписные" анимации: нервный тик, характерный жест).
- **Обвинитель/адвокат (Prosecutor/Defense):** самый полный набор — все позы §2 + Objecting +
  Pointing + сценовые one-shot из §3.
  Позднее появившиеся персонажи (Apollo Justice, Athena Cykes) получают доп. уникальные
  геймплейные анимации, завязанные на их особые механики (Perceive/браслет, анализ эмоций).
- **Второстепенные/массовые персонажи (галерея на фоне):** минимальный набор — Idle, иногда
  Talking, без Damage/Objection (используются только как "публика"/фон).

## 5. Технические слои анимации (как это устроено в спрайтах игр)

1. **Base layer** — статичный PNG/спрайт-лист позы.
2. **Mouth/Talk layer** — накладываемый цикл (2-4 кадра) движения рта, включается/выключается
   независимо от позы (то, что objection.lol выставляет через "Do not talk").
3. **Blink layer** — отдельный редкий цикл моргания, не связан с речью.
4. **Idle motion** — микро-луп (дыхание/покачивание), почти незаметный, отличает "живой" кадр
   от статичной картинки.
5. **Effect layer** (несвязан с самим спрайтом персонажа) — трещины экрана, вспышки, "!"-иконки,
   капли пота крупным планом, отдельные партиклы — накладываются поверх сцены синхронно с позой.
6. **Audio-sync markers** — многие one-shot позы (Objection, Desk Slam, Gavel) жёстко
   синхронизированы со звуковым эффектом/voice clip, а не идут отдельно от звука.

## 6. Логика стейт-машины (переходы)

Общее правило, которое использует большинство fan-инструментов (включая objection.lol) и,
судя по всему, оригинальные игры:

- Персонаж всегда находится в одной **базовой позе** между репликами.
- Начало произнесения текста → включается **Talking-оверлей** текущей позы (если не выставлен
  флаг "не говорить").
- Конец текста / пауза → Talking-оверлей выключается, персонаж возвращается в статичный вариант
  той же позы.
- Явная смена позы (по тегу в скрипте сцены) — заменяет базовую позу целиком, one-shot
  переходная анимация (если есть) проигрывается один раз, затем стейт "оседает" в новую idle-позу.
- **Damage** — отдельный триггер вне обычного потока: прерывает текущую позу, проигрывает
  shock-кадр + экранный эффект, затем обычно откатывается в Sweating/Disheartened, а не в
  прежнюю Confident-позу.
- **Merge text with next** (objection.lol) — техника, при которой одна текстовая реплика
  продолжается визуально через смену позы БЕЗ разрыва диалогового потока (смена кадра "внутри"
  одной фразы) — полезный паттерн, если нужно менять экспрессию персонажа посреди длинной строки.
