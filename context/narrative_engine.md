# DUELOGUE — Нарративный движок (v0.2 — модель осей)

**Статус:** спека нарративного слоя поверх ядра «Зал» (`zal_gdd.md`, ядро v0.3.1).
Не трогает геймдизайн — читает **лог механики** и собирает из него процедурное повествование.
**Реализация (план):** `test/prototype_zal/narrative_*.gd` + данные темы. **Источник событий:**
`_emit(...)` в `zal_v3.gd` (события `move` / `clinch` / `end`) и состояние `zal_v3_model.gd`.

---

## 1. Цель

Превратить лог битвы аргументов в **связный обмен репликами**, который ощущается как
реальный спор: смешной, реалистичный, читаемый. Так же, как механика отражает флоу дебатов,
нарратив должен отражать его на уровне речи.

Две несущие цели:
1. **Модульная контекстуальность.** Тему карт можно менять (рукой или ИИ), и структура
   нарратива не рушится. Колода — это набор **риторических приёмов** (форма); тема — это
   **данные** (содержание), на которые приёмы наводятся.
2. **Фановость.** Реплики отражают суть спора, связываются в диалог, звучат живо и забавно.

## 2. Несущий принцип: форма ≠ содержание

> **Тема приколота к РАМКЕ, не к карте. Карта — это глагол. Рамка — существительное.**

Карты атаки и тезисы **никогда не несут тему** — они несут риторическую функцию (приём).
Содержание (о чём спор) поставляют **Установки** (claim рамки) и **пул мотивов темы**.
Сменил тему → поменялись рамки-топики и пул мотивов; глагол (карта) и рисунок спора — те же.

## 3. Три дискурс-объекта + корневая правка содержания (v0.2)

Разбор/Кража бьют **не абстрактную рамку, а конкретный тезис на ней**. Рамка падает, только
когда снят последний тезис. Тезисы — это «здоровье».

**Корневая правка (см. лог 10 симуляций):** приём НЕ должен быть содержанием. В v0.1 тезис был
`приём(claim, случайный мотив)` — это пусто: рамка с 5 тезисами = одна позиция, повторённая 5
раз, а разбор грыз *имя приёма* («из „ссылка: Гавайи“ не вытекает»), а не довод. Спор шёл о
ходах, не о пицце.

**Фикс: содержание = взгляд полюса на спорную ОСЬ темы.** Приём = лишь МАНЕРА подачи.

| Объект | Чем создаётся | Содержание |
|---|---|---|
| **Рамка (headline)** | Установка | широкая позиция-якорь стойки |
| **Тезис (довод)** | Тезис: `приём(взгляд-на-ось)` | взгляд своего полюса на ось |
| *(атака)* Разбор | рефьютит конкретный тезис | **взгляд противоположного полюса на ТУ ЖЕ ось** (содержательный) либо придирка к тексту довода (процедурный) |
| *(атака)* Кража | присваивает тезис | разворот оси: «твой взгляд работает на меня» |

Так разбор отвечает реальным контр-доводом по тому же под-вопросу → pro/contra по-настоящему
сталкиваются, связность и клэш возникают сами.

## 4. Два уровня содержания + анти-взрыв

- **Рамка (headline)** — широкая позиция стойки (для Установки). Якорь-топик нити.
- **Тезис** — **взгляд на ось** (конкретный довод), подпирающий рамку.

**Почему не взрывается комбинаторно.** Развести два «количества»: *пространство диалогов* (что
игрок увидит) пусть растёт по экспоненте — это реиграбельность, его собирает движок. *Авторский
контент* должен расти **линейно**. Достигается тем, что:
- **ортогональные модификаторы** (приём-манера, дискурс-маркер, флаг коллбэка, какая рамка, шаг
  клинча, персона) — комбинируются движком, авторинг НЕ множат;
- **контент авторится атомами = спорными осями**: 1 ось = 1 под-вопрос с ДВУМЯ
  противоположными взглядами (contra/pro). Counter = взгляд противоположного полюса на ту же
  ось → отдельно не пишется. Авторинг ≈ `#осей×2 + #headline + обёртки` — линейно.

Клинч и много рамок линейности не ломают: защита в клинче берёт *свежую ось* для рамки (глубина
потребляет больше осей, а не требует цепочек контр-на-контр); атака на старую рамку = коллбэк
(ортогонально).

## 5. Контракт темы (data-схема)

Любая тема — рукописная или сгенерённая ИИ — реализует этот контракт; структура движка не
зависит от темы.

Содержание темы = **граф аргументов** (стойки с headline-позициями + спорные оси), не плоский
мешок мотивов. (Реализация: `test/prototype_zal/narrative_theme_pineapple.gd`.)

```
Theme {
  id: "pineapple",
  topic: "Ананас на пицце",
  stances: {
    contra: { label: "против ананаса",
              headlines: [ "ананасу не место на пицце", "это порча классики", ... ] },  # для Установок
    pro:    { label: "за ананас",
              headlines: [ "ананас на пицце — это прекрасно", "кухня должна быть свободной", ... ] }
  },
  axes: [                                  # ← сердце: спорные под-вопросы, по 2 взгляда
    { id: "balance",
      contra: "ананас ломает вкусовой баланс пиццы",
      pro:    "сладко-солёный контраст только обогащает вкус",
      motifs: [ "сыр", "томатный соус", "рецепторы" ] },
    { id: "tradition", contra: "это плевок в неаполитанский канон",
      pro: "кухня всегда эволюционировала, канон не догма", motifs: [ "Неаполь", "ЮНЕСКО", ... ] },
    ...  # ~6 осей закрывают весь спор
  ],
  shared_motifs: [ "здравый смысл", "TikTok-повара", ... ],
  voices: { contra: "пуристичный сноб", pro: "наглый новатор" }   # ОПЦИЯ (раздел 10), v0 — нейтр.
}
```

**Требования к корректности темы:** `headlines` каждой стойки ≥ Установок (U=3) + старт;
осей ≥ глубины самого долгого клинча/числа тезисов на рамке (иначе оси повторяются с коллбэк-
флейвором — «опять ты про баланс?»). Мотивы привязаны к оси → нет «статистика: канадский
изобретатель».

## 6. Приём = МАНЕРА подачи (не содержание)

Каждая карта = свой приём; ростер тема-независим, кластеризуется в этос/пафос/логос (задел на
ось §9 GDD). **Важно (v0.2): приём оборачивает взгляд-на-ось, но не несёт смысла сам.**

**Тезис — манеры подачи своего взгляда** (`приём(взгляд)`): Пример (логос), Авторитет (этос),
Статистика (логос), Аналогия (логос), Здравый смысл (этос), Эмоция (пафос), Определение (логос),
Традиция (этос).

**Разбор — два рода** (оба ссылаются на реальный довод оппонента, не на имя приёма):
- **Содержательные** — контент = взгляд своего полюса на ту же ось: Контрпример («как раз
  наоборот: …»), До абсурда (reductio), Ложная аналогия.
- **Процедурные** — придирка к *тексту* чужого довода, без оси: Источник? (пруфы), Передёрг
  (strawman), Не в кассу (red herring), Корреляция≠причина.

**Кража** — разворот оси (взгляд оппонента → довод за себя): Разворот, Та же логика.

> Приём фиксирован картой (её идентичность), содержание приходит из оси темы. Рескин темы
> меняет содержание спора, состав приёмов — нет. Манеру с содержанием не путать — это и был
> корень пустоты в v0.1.

## 7. Событие лога → речевой акт (детерминир., тема-агностично)

| Событие механики | Речевой акт | Операнд |
|---|---|---|
| Тезис на свою рамку | ASSERT | своя активная рамка (claim) |
| Установка (новая) | OPEN; если рушит спорную — RETREAT-PIVOT | новый claim из стойки |
| Разбор | REFUTE | целевой тезис |
| Кража | APPROPRIATE | целевой тезис |
| Тезис в клинче (защита) | HOLD | спорная рамка |
| Добив в клинче | PRESS | целевой тезис |
| Атакующий перестоял | LAND (+ DEMOLISH если рамка пала) | сорванные тезисы |
| Защитник перестоял | WITHSTAND | усиленная рамка |
| Redeploy (страховка) | SCRAMBLE | новый claim в панике |
| Пас (карт нет) | SILENCE | — |
| Нокаут / конец | VERDICT | зал |

Флаг **callback**: если цель Разбора/Кражи — закрытая (старая) рамка → маркер «возвращаясь
к…»; если активная → обычный ответный маркер.

## 8. Состояние дискурса (делает это ОБМЕНОМ)

Нарративный слой ведёт параллельную модель (лог — источник истины по числам):

```
Frame { claim(headline), stance, statements: [ Statement ] }   # стек доводов на рамке
Statement { axis, pole, device, motif, text, gist }    # axis — id оси; gist = текст взгляда (для ссылок)
Discourse {
  frames_you[], frames_opp[],     # зеркало линий рамок
  contested: Frame?,              # текущая спорная нить (клинч)
  last_line, last_speaker,        # для ответных маркеров
  used_motifs{},                  # анти-повтор + коллбэки
  tone: zal() → "уверенный"|"ровно"|"на отчаянии"|"заискивает"
}
```

Управление от лога:
- Тезис → **push** Statement в стек активной рамки стороны.
- Разбор landed (N) → **pop** N доводов цели (LIFO: рефьютим последний сказанный).
- Кража → pop у защитника + push «перехват» на активную рамку атакующего.
- стек опустел → рамка снята (DEMOLISH).
- клинч → `contested` = атакованная рамка; вся перепалка идёт по её топику.

`gist` каждого Statement позволяет последующей атаке сослаться на реальную реплику.

## 9. Формула сборки (Realize) — через ось

```
line = Realize(act, device, operand, discourse, voice)
```

Сборка по типам карт (`pole(side)` = стойка стороны; `axis[pole]` = взгляд полюса на ось):
- **Установка** → `next_headline(side)` → реплика, открывающая нить.
- **Тезис** → выбрать свежую ось (не звучавшую на рамке) → `take = axis[pole]` → `device(take, мотив-оси)`
  → Statement{axis, gist=take, text}. push в стопку.
- **Разбор** → берёт `axis` целевого Statement:
  - содержательный приём → контент = `axis[pole(attacker)]` (взгляд своего полюса на ту же ось);
  - процедурный приём → оперирует над `operand.gist` (текст чужого довода).
- **Кража** → разворот: `«operand.gist»? → axis[pole(attacker)]` («это и есть довод за меня»).

Фолбэк: если у цели нет оси (голый headline) — берётся свежая ось, gist = headline. *(Открытая
нить: при краже голого headline ось случайна и связь слабее — кандидат на контекстный выбор.)*

**Дискурс-маркеры** (тема-независимы, выбор по контексту):
| Контекст | Маркер (примеры) |
|---|---|
| OPEN | «Объявляю:…», «Вот моя позиция:…» |
| ASSERT после хода оппонента | «И вообще,…», «Более того,…» |
| REFUTE активной рамки | «Да брось,…», «Секунду,…» |
| REFUTE старой рамки (callback) | «Кстати, возвращаясь к…» |
| HOLD (защита в клинче) | «Стоп-стоп,…», «Это не отменяет, что…» |
| RETREAT-PIVOT | «Ладно, забудем. Вот что важно:…» |
| SCRAMBLE | «Хорошо, хорошо, но послушайте…» |
| тон «на отчаянии» | заискивание перед залом, нервозность |

## 10. Мастер-колода: fill-safe скелет + контракт темы (LLM = офлайн-автор тем)

Рантайм — **шаблонный** (дёшево, воспроизводимо). LLM не генерит реплики вживую; LLM —
**офлайн-автор данных темы**. Грамматику и «живость» гарантирует то, что скелет спроектирован
**fill-safe**: любая корректно заполненная тема собирается без кривых конструкций.

**Мастер-колода = три вещи:**
1. **Фиксированный fill-safe скелет** (`narrative_engine.gd`) — маркеры, обёртки-приёмы, точки
   склейки. Языково-грамотный, тема-независимый, пишется один раз.
2. **Контракт темы** (ниже) — по-полю ограничения, по которым ИИ генерит новую колоду.
3. **Эталонные темы** — `narrative_theme_{pineapple,shawarma,evangelion}.gd` (одна структура,
   разный фантик; evangelion сгенерирована по контракту §10 — доказывает цель «ИИ делает колоды».
   Движок не меняли ни для одной).

**Несущее правило fill-safe (грамматика by construction):**
> Движок **НИКОГДА не склоняет** вставленное значение. Слот бывает двух видов:
> 1. **целая фраза** (headline/take/gist) — вставляется на границе предложения (после маркера,
>    точки, тире, в кавычках);
> 2. **существительное в именительном** (motif) — только в именительно-безопасной позиции
>    (приложение «…, например», сравнение «как …»). Падежных слотов (`спроси {x}`, `у {x}`,
>    `с {x}`) в паттернах НЕТ.

Манера приёма живёт в связке про **взгляд**, а не про мотив (нет «Статистика: здравый смысл»).

**Контракт полей темы (то, что заполняет ИИ):**
| Поле | Форма | Правило |
|---|---|---|
| `stances.{contra,pro}.label` | род. оборот | «за обычный лаваш» |
| `headlines[]` | целая фраза, наст. время, строчная, без точки | широкая позиция |
| `axes[].id` | короткий лат. id | уникален |
| `axes[].{contra,pro}` | целая фраза-взгляд (как headline) | прямо противоположны по одной оси |
| `axes[].motifs[]` | **именительный** нос. (1–3 слова) | сущность, на которую можно сослаться |
| `shared_motifs[]` | то же | общий резерв |

Достаточность: `headlines` ≥ U+1; осей ≥ глубины клинча. «Живость» — в остроумии фраз `take`/
`headline` (их и пишет ИИ), безопасную сборку даёт скелет.

**Чек-лист генерации колоды (промпт ИИ):** дать тему и две стороны → попросить 6 осей (по
взгляду на сторону, прямо противоположных) + по 6 headline на сторону + по 2–3 именительных
мотива на ось; запретить точки/падежные формы в `take`; держать фразы короткими и колкими.

*(Поздний refinement: тег `appeal` (логос/пафос/этос) у оси → биас выбора приёма-манеры, чтобы
«Статистика»/«Эмоция» точнее ложились. Это же — ось §9 GDD.)*

## 13. Критический разбор v0.2 по живым стенограммам (2026-07-02) → план v0.3

Логичность достигнута (реплики связны, клэш по осям реален), но флоу дебатов не дожат.
Три диагноза (примеры — `narrative_transcript.md`, матч 1782989897):

**Д1. Приём слишком тонок — перекоррекция v0.2.** v0.1: приём = содержание (пусто).
v0.2: приём = чистая обёртка → взгляд-на-ось несёт ~90% строки, приём — генерический
хвостик, иногда противоречащий содержанию («Называть это пиццей — подмена понятий.
Цифры на моей стороне.» — Статистика без цифр; Традиция на анти-традиционном взгляде).
Приёмы взаимозаменяемы по построению → не ощущаются. Плюс громоздкое цитирование:
каждая атака тащит полный gist в кавычках, «До абсурда» — три полных фразы в строке.
Фиксы (fill-safe, авторинг линеен):
- **Тег оси**: контракт темы +`axes[].tag` (имен., 1–2 слова: «канон», «баланс»).
  Полная цитата — только первое упоминание; дальше — тегом («Опять ты со своим каноном?»).
  Процедурный клип длинных цитат по границе слова + «…».
- **Appeal-биас** (из §10): тег логос/пафос/этос у оси → приём выбирает подходящую ось;
  мисматчи «Статистика без цифр» исчезают.
- **Приём = ФОРМА предложения, не хвостик**: Статистика — цифровой заход («девять из
  десяти…»), Здравый смысл — риторический вопрос, Эмоция — восклицание с обрывом,
  Аналогия — сравнение через мотив. Скелет переписывается один раз.

**Д2. Монотонная длина** (все строки 8–15 слов, один ритм). Фикс — **регистры**
full/short/punch у каждого приёма (линейно по приёмам) + НАПРАВЛЕННЫЙ выбор:
- клинч = аччелерандо: 1-й удар полный, 2-й короткий с тегом, 3-й+ панч («Пруфы!»,
  «Мимо.», «Ещё!») — драйвер уже есть (r_count/t_added);
- тон от zal(): отстающий коротко-нервно, фаворит вальяжно (реализует tone из §8);
- большие события (захват «со всей силой», счёт судьи) — длинные тирады.

**Д3. Психатаки** (сарказм, ироничное молчание, жесты) — парковка, НО два шага дёшевы
уже сейчас: (а) пас при ведущем зале = «(выразительно молчит и смотрит в зал)» вместо
«сказать нечего» — один if по zal(); (б) сарказм = punch-регистр процедурного разбора
(«"Канон", ага.»). Жесты-ремарки — канал под character_core (шина utterance готова).
Полноценные карты-психатаки — после регистров, отдельным слоем.

**Порядок v0.3:** (1) регистры + клинч-аччелерандо + ироничное молчание; (2) тег оси +
сжатие цитат; (3) appeal-биас + формы-предложения приёмов. Каждый шаг проверяется
narrative_smoke + чтением стенограммы (метрика — ср. длина строки по контекстам и
отсутствие мисматчей приём/содержание).

## 14. ФОРМУЛЬНОЕ ЯДРО (v0.3) — риторика × геймдизайн × генерация

Фундамент под тактики §13: набор формул, где у каждого слота реплики есть (а) драйвер из
геймдизайна, (б) якорь в риторическом корпусе `context/rhetoric/`, (в) линейная стоимость
авторинга, (г) критерий для генерации тем по мастер-шаблону.

### 14.1 Главная формула разделения (модульность)

> **СПОР = СКЕЛЕТ(риторика) × ТЕМА(позиция)**

- **Скелет** — схемы аргументов, критические вопросы, регистры, маркеры, референция.
  Пишется ОДИН раз, тема-независим. Источник — риторический корпус (он универсален
  по построению: `_summary_universal_methods.md` отбирал приёмы по пересечению школ).
- **Тема** — оси-стазисы, взгляды, теги, мотивы, headlines. Генерится по контракту §5/§10.

Тест модульности двусторонний: смена темы не меняет ни строки скелета (доказано 3 темами)
И улучшение скелета не требует трогать темы (проверять при каждом шаге v0.3).

### 14.2 Формула реплики — четыре ортогональных слота

> **Реплика = Маркер(акт, фаза) + Референция(цель, история) + Форма(схема|CQ, регистр)⟨Содержание(ось, полюс)⟩**

| Слот | Что решает | Драйвер (геймдизайн) | Якорь (риторика) | Авторинг |
|---|---|---|---|---|
| Маркер | место в диалоге | событие лога (§7) + фаза (14.7) | дискурс-маркеры | ~30 строк, 1 раз |
| Референция | как ссылаюсь на чужое | история упоминаний оси | анафора/дейксис | политика, 0 строк |
| Форма | как звучит приём | карта + регистр (14.5) | схемы Уолтона + CQ | #схем×формы, 1 раз |
| Содержание | о чём говорю | доска (рамка/ось/полюс) | стазисы (оси) | тема, линейно |

### 14.3 Тезис = аргументационная схема (Уолтон)

> **Тезис(карта) = Схема_карты(взгляд(ось, полюс), мотив(ось))**

Приём тезиса — не «хвостик», а СХЕМА, задающая форму предложения целиком:

| Приём | Схема Уолтона | appeal | Форма (пример скелета) |
|---|---|---|---|
| Авторитет | Expert Opinion (#1) | этос | «спроси любого, кто в теме: {t}» |
| Пример | from Example (#8) | логос | «{t}. Один {x} чего стоит.» |
| Аналогия | from Analogy (#9) | логос | «{t} — та же история, что с {x}» |
| Статистика | Correlation/Popular | логос | «девять из десяти скажут: {t}» |
| Традиция | Precedent/Popularity | этос | «{t} — не мы придумали, не нам отменять» |
| Здравый смысл | Position to Know | этос | «да ты сам знаешь: {t}» |
| Эмоция | from Consequences | пафос | «{t} — и мне не всё равно!» |
| Определение | from Classification | логос | «по определению: {t}» |

Statement помнит `{scheme, axis, gist}` — **схема становится ПОВЕРХНОСТЬЮ ДЛЯ АТАКИ**.

### 14.4 Разбор = критический вопрос к схеме цели ← КОРНЕВАЯ ФОРМУЛА v0.3

> **Разбор(карта, цель) = CQ(схема цели, тип карты) | Контр-взгляд(ось цели) | Придирка(текст цели)**

Уолтон: у каждой схемы есть стандартные критические вопросы — готовые пары «удар↔контра».
Приоритет резолва: если тип атакующей карты попадает в CQ-слот схемы цели → CQ-форма
(атака, заточенная под форму цели); иначе — содержательный/процедурный путь как сейчас.

Примеры (одна и та же карта звучит по-разному от того, ЧТО бьёт — вот она «уникальность»):
- «Источник?» по тезису-Авторитету → CQ#1 Expert Opinion: «а твои "знающие люди" — это кто конкретно?»
- «Источник?» по тезису-Статистике → CQ выборки: «чья статистика? кто считал?»
- «Ложная аналогия» по Аналогии → CQ#2 Analogy: «сравнение хромает: {x} — это не {tag}»
- «Контрпример» по Примеру → CQ репрезентативности: «один случай — не правило»
- «До абсурда» по Традиции → «раньше и {x} нормой считали»

Авторинг: #схем(8) × ~3 CQ ≈ 24–30 строк скелета, ОДИН раз, тема не участвует. Линейно.
Геймплейный смысл: появляется невидимая в механике, но слышимая в речи «система попаданий»
— атака в форму цели звучит хлёстко, мимо формы — генерично. Задел под именные приёмы (§9 GDD).

### 14.5 Накал → регистр (динамика длины)

> **Накал = w₁·глубина_клинча + w₂·|зал|/K + w₃·счёт_судьи + w₄·фаза**
> **Регистр(накал) ∈ {тирада, полный, короткий, панч, молчание}**

Клинч = аччелерандо (1-й удар полный → панчи «Пруфы!»); пики (захват «со всей силой»,
тик счёта судьи) форсируют тираду; пас при ведущем зале = ироничное молчание. Все драйверы
уже есть в модели (r_count, zal(), crowd_streak, доля колоды) — формула только читает их.

### 14.6 Референция (сжатие цитат)

> **Реф(ось, n_упоминаний) = полная цитата (n=0) → клип «…» (1) → тег (2+) → нулевая («опять ты за своё»)**

Контракт темы +`axes[].tag` (имен., 1–2 слова: «канон», «баланс»). Дискурс ведёт счётчик
упоминаний оси. Чем дольше спор — тем короче ссылки: живая речь так и работает.

### 14.7 Фаза дебата

> **Фаза(доля_колоды) ∈ {дебют, разгар, эндшпиль}**

Дебют — формальные заявления («Объявляю:»), разгар — столкновение, эндшпиль — апелляция
к залу («судите сами», обращение к публике — ровно когда зал-механика решает исход).
Якорь: структура реальных дебатов (opening/rebuttal/closing). ГД-драйвер уже читаем
физически: кап колоды = таймер (GDD §7).

### 14.8 Апелляции и дрейф (Аристотель)

> **P(appeal) = база(приём) + дрейф(зал)**

appeal(схемы) — колонка в 14.3; appeal(оси) — тег темы. Биас: приём предпочитает ось
своей апелляции (уходят мисматчи «Статистика без цифр»). **Дрейф:** отстающий по залу
смещается к пафосу, фаворит — к этосу (наблюдение реальных дебатов: проигрывающий
эмоционирует). Тон партии меняется сам, из механики.

### 14.9 Критерии компилируемости темы (мастер-шаблон → executable)

Каждый критерий механически проверяем → цель: линтер `tools/theme_lint.gd`.
Пайплайн генерации: промпт по контракту → линт → smoke-стенограмма → чтение.

| # | Критерий | Проверка |
|---|---|---|
| К1 | Полнота: осей ≥ 6, headlines ≥ U+1 | счёт |
| К2 | Оппозиция: contra ⊥ pro по ОДНОМУ вопросу | негейт-тест (вручную/LLM-судья) |
| К3 | Fill-safe: взгляды — целые фразы без точки; мотивы — имен. 1–3 слова | regex |
| К4 | Мотив конкретен: проходит «например»-тест («…, например {x}») | эвристика + чтение |
| К5 | Тег оси: 1–2 слова, имен. | regex |
| К6 | Appeal-покрытие: среди осей ≥1 логос, ≥1 пафос, ≥1 этос | счёт по тегам |
| К7 | Headline ≠ take дословно (рамка шире оси) | строковая дистанция |

### 14.10 Порядок реализации v0.3 (пересматривает §13)

Формулы меняют приоритет: корень «приёмы не чувствуются» — не в регистрах, а в 14.3–14.4.

1. **Схемы + CQ** (14.3–14.4): Statement.scheme, CQ-таблица, формы-предложения тезисов.
2. **Референция + тег** (14.6): `axes[].tag` в контракт и в 3 темы, счётчик упоминаний.
3. **Накал + регистры + фаза** (14.5, 14.7): аччелерандо клинча, ироничное молчание, тирады пиков.
4. **Апелляции + дрейф** (14.8): appeal-теги осей в темы, биас+дрейф выбора.
5. **theme_lint** (14.9) — в любой момент, независим.

Каждый шаг: narrative_smoke → чтение стенограммы → фиксация в §13-метриках (длина строки
по контекстам, мисматчи, доля CQ-попаданий).

## 15. КАСКАД СТОЛКНОВЕНИЙ → СЕМАНТИЧЕСКАЯ СТРУКТУРА (черновик, НА ОБСУЖДЕНИИ)

Задача (от игрока): плоские CQ-пары «удар↔контра» недостаточны для семантической
эмерджентности; нужен расчёт каскада взаимоотношений карт и структура, дающая баланс
между числом вариаций (карта × тема) и естественными следствиями/завершениями реплик.
Найденная проблема мысленного эксперимента: чистый CQ истончает клэш осей.

### 15.1 Расчёт каскада (что с чем реально сталкивается)

| Порядок | Столкновение | Сырое пространство | Кто его порождает |
|---|---|---|---|
| 0 | Тезис × ось × полюс | 8 схем × 6 осей × 2 = 96 звучаний | доска (какая рамка активна) |
| 1 | Атака × схема цели | (7Р+2К) × 8 схем = 72 пары | выбор цели атаки |
| 2 | Защита × последний удар | 8 схем × 7 хватов = 56 переходов | рука защитника (экономика!) |
| 3 | Добив × последняя защита | = порядок 1 рекурсивно | ралли клинча (r_count) |
| 4 | Резолв × последний неотвеченный ход | 7 × 3 исхода = 21 | механика «кто перестоял» |

Сырой авторинг каскада = 72+56+21 ≈ 150 КВАДРАТИЧНЫХ пар — запрещено анти-взрывом.
Решение: факторизовать все три матрицы через ОДИН общий словарь.

### 15.2 Словарь ЗАЦЕПОК (7 семантических валентностей, из CQ Уолтона)

**ЗАЦЕПКА** — то, за что атака хватает схему: ИСТОЧНИК, ИСКЛЮЧЕНИЕ, СХОДСТВО,
СЛЕДСТВИЕ, ПОДМЕНА, СВЯЗЬ, УМЕСТНОСТЬ. Один словарь играет ТРИ роли:

1. **Хват (порядок 1):** атака-карта несёт одну зацепку (Источник?→ИСТОЧНИК,
   Контрпример→ИСКЛЮЧЕНИЕ, Ложная аналогия→СХОДСТВО, До абсурда→СЛЕДСТВИЕ,
   Передёрг→ПОДМЕНА, Корреляция→СВЯЗЬ, Не в кассу→УМЕСТНОСТЬ; Кража — не зацепка,
   а ПРИСВОЕНИЕ оси). Схема тезиса ОТКРЫВАЕТ 2–3 зацепки (таблица 15.3).
   Зацепка ∈ открытых → HIT (заточенная реплика), нет → MISS (generic, как сейчас).
2. **Ответ (порядок 2, adjacency pair):** у зацепки есть ОТВЕТНЫЕ схемы —
   защита картой ответной схемы = РЕПЛИКА-ОТВЕТ («Кто считал? — Вот кто: …»),
   любой другой картой = РЕПЛИКА-УКЛОНЕНИЕ («Не в цифрах дело — дело в каноне»).
   Оба хода легитимная риторика; звучат по-разному. ЭКОНОМИКА РУКИ НАЧИНАЕТ
   ГОВОРИТЬ: у кого нет ответной карты — тот слышимо юлит.
3. **Закрытие (порядок 4):** резолв ссылается на последнюю НЕотвеченную зацепку:
   «источника зал так и не услышал» / «сравнение так и осталось хромым» /
   (защитник перестоял) «на каждый вопрос нашёлся ответ».

### 15.3 Онтология скелета (все декларации, счёт)

**Схема тезиса (8):** форма подачи (×2-3 варианта ×3 регистра) + **ОБЪЕКТ** (уязвимое
слово: эксперты/цифры/случай/сравнение/традиция/очевидность/чувства/термин) +
открытые зацепки + appeal.

| Схема | Объект | Открытые зацепки |
|---|---|---|
| Авторитет | эксперты | ИСТОЧНИК, УМЕСТНОСТЬ, ИСКЛЮЧЕНИЕ |
| Статистика | цифры | ИСТОЧНИК, СВЯЗЬ, ИСКЛЮЧЕНИЕ |
| Пример | случай | ИСКЛЮЧЕНИЕ, ИСТОЧНИК, СХОДСТВО |
| Аналогия | сравнение | СХОДСТВО, ИСКЛЮЧЕНИЕ |
| Традиция | традиция | СЛЕДСТВИЕ, УМЕСТНОСТЬ |
| Здравый смысл | очевидность | ИСТОЧНИК, СЛЕДСТВИЕ |
| Эмоция | чувства | УМЕСТНОСТЬ, ПОДМЕНА |
| Определение | термин | ПОДМЕНА, СЛЕДСТВИЕ, ИСТОЧНИК |

**Зацепка (7):** хват-шаблон со слотом объекта («а {объект} — чьи/откуда?») +
ответные схемы (~2) + шаблон уклонения (1 общий) + закрывашка.
**Ключевая факторизация:** хват-шаблон × объект схемы → 7×8=56 РАЗЛИЧИМЫХ звучаний
из 15 деклараций («а эксперты — кто?» vs «а цифры — откуда?» — один шаблон!).

Счёт: hit-пар = 20 из 56 (~35% — «попадание» остаётся событием). Авторский объём
скелета: ~50 форм тезисов + 7 хватов + ~14 ответов + 7 закрывашек + обёртки ≈ **120–150
строк, ОДИН раз, тема-независимо**. Различимых диалоговых рисунков (схема×ось×хват×
ответ/уклон×регистр×референция) — порядка 10⁴. Тема добавляет ЛИНЕЙНО: 12 взглядов+теги.

### 15.4 Двухуровневая структура спора (лечит истончение оси)

> **КЛИНЧ = СТАЗИС(ось) + ОБМЕН(цепочка зацепок)**

Ось атакованного тезиса = ТЕМА раунда (все реплики ралли держат её тег: «Так это
канон не догма?» … «Канон, канон»). Зацепки = ТАКТИКА внутри темы. Клэш взглядов
живёт на уровне стазиса (тезисы, содержательные разборы, кража-присвоение),
пикировка — на уровне обмена. Ось не истончается: она рамка, в которой звучат CQ.
Правило регистра из §13-эксперимента: полный регистр = хват+контр-взгляд,
короткий/панч = чистый хват.

### 15.5 Обмен как автомат (естественные следствия и завершения)

Стек обмена: PRESS кладёт зацепку → HOLD отвечает (снимает) или уклоняется (висит)
→ RESOLVE закрывает последнюю висящую (текст закрывашки) либо фиксирует «все сняты».
Свойства: (а) реплики связаны попарно (вопрос→ответ/уклон) — диалог когерентен по
построению; (б) финал ралли семантически вытекает из него; (в) рисунок цепочки
порождается РУКАМИ игроков, не автором — эмерджентность из геймплея.

### 15.6 Открытые вопросы (решить до кода)

1. Баланс слотов: СХОДСТВО/СВЯЗЬ цепляют по 1-2 схемы («снайперские» хваты) vs
   ИСТОЧНИК — 5 («универсальный»). Оставить как текстуру (обучаемо!) или выровнять?
2. Уклонение — норма (нет ответной карты в руке), не должно читаться поражением;
   маркер pivot, не оправдания. Проверить мысленным экспериментом раунд 2.
3. Объект схемы можно опционально флейворить темой позже (мотив оси вместо
   генерического объекта) — парковка, сначала генерический.
4. Кража в терминах обмена: присвоение закрывает ВСЕ висящие зацепки разом
   («и спор ваш теперь мой») — проверить звучание.
